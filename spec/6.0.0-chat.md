# 6.0.0 ŌĆö Chat & Real-time Streaming

## 6.1.0 Overview

The chat interface provides real-time interaction with Claude agents via Server-Sent Events (SSE). User messages go through REST; agent responses stream back via SSE.

## 6.2.0 Communication Pattern

```
User action  ŌåÆ  POST /api/sessions/[id]/events  ŌåÆ  Anthropic API
Agent stream ŌåÉ  GET  /api/sessions/[id]/stream   ŌåÉ  Anthropic API (SSE)
```

Bidirectional communication without WebSocket ŌĆö REST for user actions, SSE for agent responses.

## 6.3.0 Event Types

### 6.3.1 User ŌåÆ Agent (sent via POST)

| Event | Description |
|-------|-------------|
| `user.message` | Text message from user |
| `user.interrupt` | Stop current agent response |
| `user.custom_tool_result` | Result for custom tool call |
| `user.tool_confirmation` | Confirm/deny tool execution |

### 6.3.2 Agent ŌåÆ User (received via SSE)

| Event | Description |
|-------|-------------|
| `agent.message` | Text response (streamed token-by-token) |
| `agent.thinking` | Extended thinking content |
| `agent.tool_use` | Tool invocation (name + input) |
| `agent.tool_result` | Tool execution output |
| `agent.mcp_tool_use` | MCP tool invocation |
| `agent.mcp_tool_result` | MCP tool output |
| `agent.custom_tool_use` | Custom tool request |
| `session.status_idle` | Agent finished processing |
| `session.status_running` | Agent started processing |

## 6.4.0 SSE Streaming Implementation

1. Frontend opens `EventSource` to `/api/sessions/[id]/stream`
2. Server route creates Anthropic SDK stream
3. Stream is proxied through SvelteKit's `ReadableStream`
4. Client parses SSE events and updates UI reactively

## 6.8.0 Design Decision: SSE over WebSocket

SSE fits the agent communication model:
- Agent responses are unidirectional streams (server ŌåÆ client)
- User actions are discrete events (client ŌåÆ server via REST)
- SSE is simpler to implement, debug, and proxy
- No need for full-duplex communication
