# 8.0.0 — Settings & Preferences

## 8.1.0 Overview

Settings is split by role. All users manage personal preferences (theme, defaults, password). Admins additionally manage the global API key, user accounts, invitations, SMTP configuration, and OAuth provider credentials for MCP services.

Settings uses a sidebar-nav layout: tabs on the left, content on the right, with a persistent user-identity footer at the bottom of the sidebar containing the logout action (see 8.8.0).

## 8.2.0 Data Model

### 8.2.1 API Keys (Global)

A single global API key is shared across all users. Only admins can set, rotate, or delete it.

| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| user_id | uuid | FK → users (admin who set it) |
| name | text | Always `'Global'` |
| encrypted_key | bytea | AES-256-GCM encrypted |
| iv | bytea | Initialization vector |
| label | text | Optional display label |
| created_at | timestamp | |
| updated_at | timestamp | |

### 8.2.2 API Key Shares (Future)

Prepared for future per-user key sharing. Not yet exposed in UI.

| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| api_key_id | uuid | FK → api_keys (cascade) |
| shared_with_user_id | uuid | FK → users (cascade) |
| created_at | timestamp | |

Unique constraint on `(api_key_id, shared_with_user_id)`.

### 8.2.3 User Preferences

| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| user_id | uuid | FK → users, unique (cascade delete) |
| theme | text | `'light'` or `'dark'` |
| default_agent_id | text | Optional default agent |
| default_env_id | text | Optional default environment |
| created_at | timestamp | |
| updated_at | timestamp | |

### 8.2.4 SMTP Settings

Single-row table for email configuration. Password encrypted with AES-256-GCM.

| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| host | text | SMTP server hostname |
| port | integer | 587 (STARTTLS) or 465 (implicit TLS) |
| username | text | SMTP username |
| encrypted_password | bytea | AES-256-GCM encrypted |
| iv | bytea | Initialization vector |
| from_email | text | Sender address |
| updated_by | uuid | FK → users (admin who last updated) |
| updated_at | timestamp | |

## 8.3.0 API Routes

### 8.3.1 API Key (Admin Only)

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/settings/api-key` | Check if global API key exists |
| POST | `/api/settings/api-key` | Set or replace the global API key |
| DELETE | `/api/settings/api-key` | Remove the global API key |

### 8.3.2 Admin — User Management

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/admin/users` | List all users (email, role, created date) |
| DELETE | `/api/admin/users` | Delete a user by ID (cannot delete self, cascades) |

### 8.3.3 Admin — Invitations

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/admin/invites` | List all invites with status (pending/accepted/expired) |
| POST | `/api/admin/invites` | Create invite (email, optional temp password); sends email if SMTP configured |
| DELETE | `/api/admin/invites` | Revoke a pending invite by ID |

### 8.3.4 Admin — SMTP Configuration

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/admin/smtp` | Get SMTP config (password masked) |
| PUT | `/api/admin/smtp` | Save SMTP settings (password encrypted at rest) |
| POST | `/api/admin/smtp` | Send test email to verify configuration |

### 8.3.5 Admin — OAuth Provider Configs

Per-service OAuth client credentials used by the Connections flow. See [10.0.0](./10.0.0-connections.md).

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/admin/oauth-providers` | List configured providers (client IDs masked) |
| POST | `/api/admin/oauth-providers` | Upsert client_id + client_secret (+ optional scopes override) for a service |
| DELETE | `/api/admin/oauth-providers?serviceId=…` | Remove a provider config |

## 8.4.0 Pages

| Route | Role | Purpose |
|-------|------|---------|
| `/settings` | All | Tabbed settings page: API Keys, Appearance, Account |
| `/settings` (Users tab) | Admin | User list, invite form, invite management |
| `/settings/admin` | Admin | Dedicated admin page (SMTP config) |
| `/settings/reset-password` | Auth | Forced password reset (for temp password invites) |

### 8.4.1 Settings Page Tabs

The `/settings` page uses a sidebar-nav layout:

| Tab | Audience | Contents |
|-----|----------|----------|
| API Keys | Admin | Set/rotate/delete the global Anthropic API key |
| Appearance | All | Theme toggle (light/dark) |
| Account | All | Email display, password change |
| Users | Admin | Invite new users, list existing users and invites, revoke invites, remove users |
| OAuth Providers | Admin | Configure client_id / client_secret per MCP service (see [10.0.0](./10.0.0-connections.md)) |

The logout control previously lived inside the Account tab's Session sub-section and in the top nav. It now sits in the sidebar footer (see 8.8.0).

## 8.5.0 Features

- **Global API key** (admin): Single system-wide key used by all users for Anthropic API calls
- **API key rotation** (admin): Replace encrypted key without disrupting users
- **Theme toggle**: Dark/light mode persisted in `user_preferences`
- **Default agent**: Pre-select agent when creating new sessions
- **Default environment**: Pre-select environment when creating new sessions
- **User management** (admin): View all users, delete accounts (cascades all data)
- **Invite system** (admin): Email-based or manual URL invites with 48-hour expiry
- **SMTP configuration** (admin): Configure email sending for invite delivery, with test email verification

## 8.6.0 Theme System

- Theme preference stored in DB (`user_preferences.theme`)
- Applied via `[data-theme]` attribute on `<html>` element
- Svelte store (`theme.ts`) syncs client state
- Default: dark theme

## 8.7.0 Email Service

Optional SMTP integration for sending invite emails. Implemented as a raw socket SMTP client supporting:
- **STARTTLS** (port 587)
- **Implicit TLS** (port 465)
- AUTH LOGIN authentication
- HTML email content

If SMTP is not configured, invites fall back to manual URL sharing via the admin UI. The email service is used exclusively for invite delivery.

## 8.8.0 Logout Placement

Logout is mounted as a persistent footer at the bottom of the settings sidebar, below the user-identity block (email + role). It is **not** in the top `Nav` and **not** under an Account sub-section — both of those used to host it and the duplication was confusing.

- Ghost-styled (transparent) by default
- Turns red on hover/focus to signal destructiveness
- Always visible while Settings is open — no scrolling required to reach it
