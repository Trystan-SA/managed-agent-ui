# 4.0.0 — Environment Management

## 4.1.0 Overview

Environments are sandboxed execution contexts for agent sessions, managed via the Anthropic API. They define networking rules and available packages.

## 4.2.0 Data Model

### 4.2.1 Anthropic-Owned (Source of Truth)

Environments in the Anthropic API include: id, name, networking mode, allowed hosts, installed packages, status.

### 4.2.2 Local Bookmarks

| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| user_id | uuid | FK → users (cascade delete) |
| env_id | text | Anthropic environment ID |
| nickname | text | Optional local name |
| pinned | boolean | Default false |
| created_at | timestamp | |

Bookmarks are **user-scoped** — each user has their own nicknames and pins. Deleting a user cascades to their bookmarks.

## 4.3.0 API Routes

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/environments` | List all environments |
| POST | `/api/environments` | Create new environment |
| GET | `/api/environments/[id]` | Retrieve single environment |
| PUT | `/api/environments/[id]` | Update environment |
| POST | `/api/environments/[id]/archive` | Archive environment |
| DELETE | `/api/environments/[id]/delete` | Permanently delete |

## 4.4.0 Pages

| Route | Purpose |
|-------|---------|
| `/environments` | List all environments |
| `/environments/new` | Create form (name, networking, packages) |
| `/environments/[id]` | Detail view: config, linked sessions |
| `/environments/[id]/edit` | Edit environment settings |

## 4.5.0 Networking Configuration

Two modes:
- **Unrestricted** — Full internet access
- **Limited** — Only allowed hosts (whitelist)

## 4.6.0 Package Management

Supported package managers:
- pip (Python)
- npm (Node.js)
- apt (System packages)
- cargo (Rust)
- gem (Ruby)
- go (Go modules)

Packages are installed in the environment sandbox and available to agents during sessions.
