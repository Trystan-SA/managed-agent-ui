## 9.3.0 Design Tokens

### 9.3.1 Colors

Base palette (light theme ‚Äî the default):

| Token | Value | Purpose |
|-------|-------|---------|
| `--color-bg` | `#f8f9fb` | Page background |
| `--color-surface` | `#ffffff` | Card/panel surfaces |
| `--color-surface-2` | `#f1f3f5` | Subtle surface variant |
| `--color-border` | `#e2e4e9` | Borders and dividers |
| `--color-text` | `#1a1a2e` | Primary text |
| `--color-text-dim` | `#5a5a74` | Secondary text |
| `--color-text-muted` | `#8888a4` | Tertiary/placeholder text |
| `--color-primary` | `#6366f1` | Primary actions (indigo) |
| `--color-success` | `#22c55e` | Success states (green) |
| `--color-warning` | `#f59e0b` | Warning states (amber) |
| `--color-danger` | `#ef4444` | Destructive actions (red) |
| `--color-info` | `#14b8a6` | Informational (teal) |

Dark theme overrides invert surfaces/text via `[data-theme="dark"]`.

### 9.3.2 Spacing Scale

`--space-1` (2px) through `--space-12` (64px) ‚Äî consistent spacing scale.

### 9.3.3 Typography

- **Sans**: Inter
- **Mono**: JetBrains Mono
- Type scale via `--text-xs` through `--text-2xl`

### 9.3.4 Other Tokens

- Border radii: `--radius-sm` (4px) to `--radius-full` (9999px)
- Shadows: `--shadow-sm`, `--shadow-md`, `--shadow-lg`
- Surface colors, border colors, text colors ‚Äî all tokenized

## 9.4.0 Theming

- **Light theme is the default** (base tokens define the light palette)
- Dark theme via `[data-theme="dark"]` selector overrides
- All components use tokens ‚Äî no hardcoded colors
- Theme toggle persisted in `user_preferences` table
- New users get light theme on first login

## 9.5.0 Mixins

Reusable patterns in `_mixins.scss`:
- **Card**: Consistent card surfaces with border + shadow
- **Input**: Form input styling
- **Button**: Button variants (primary, secondary, danger)
- **Truncate**: Text overflow ellipsis
- **Responsive**: Breakpoint media queries

## 9.6.0 Component Styles

One SCSS partial per component type. Components `@use` only the tokens and mixins they need ‚Äî no global style leakage.

## 9.8.0 Design Principles

- **DRY**: All values defined once in tokens, shared patterns in mixins
- **No runtime cost**: SCSS compiled at build time, CSS custom properties for theming
- **Component isolation**: Each component imports only what it needs
- **Semantic tokens**: Named by purpose (`--color-danger`), not value (`--color-red`)
