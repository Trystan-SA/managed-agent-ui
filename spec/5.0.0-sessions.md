# 5.0.0 — Session Management

## 5.1.0 Overview

Sessions are conversations between a user and a Claude agent running in a specific environment. The Anthropic API manages session state; the dashboard provides listing, creation, and lifecycle management.

## 5.2.0 Data Model

### 5.2.1 Anthropic-Owned (Source of Truth)

Sessions in the Anthropic API include: id, agent ID, environment ID, status, agent version, created/updated timestamps.

### 5.2.2 Local Bookmarks

| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| user_id | uuid | FK → users (cascade delete) |
| session_id | text | Anthropic session ID |
| nickname | text | Optional local name |
| pinned | boolean | Default false |
| created_at | timestamp | |

Bookmarks are **user-scoped** — each user has their own nicknames and pins. Deleting a user cascades to their bookmarks.

## 5.3.0 Session Status

| Status | Description |
|--------|-------------|
| `idle` | Waiting for user input |
| `running` | Agent is processing/responding |
| `rescheduling` | Session being rescheduled |
| `terminated` | Session ended |

## 5.4.0 API Routes

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/sessions` | List all sessions |
| POST | `/api/sessions` | Create new session (agent + env) |
| GET | `/api/sessions/[id]` | Retrieve session details |
| POST | `/api/sessions/[id]/archive` | Archive session |
| DELETE | `/api/sessions/[id]/delete` | Permanently delete |
| POST | `/api/sessions/[id]/events` | Send event (message, interrupt) |
| GET | `/api/sessions/[id]/stream` | SSE stream (see [6.0.0](./6.0.0-chat.md)) |

## 5.5.0 Pages

| Route | Purpose |
|-------|---------|
| `/sessions` | List all sessions with status badges |
| `/sessions/[id]` | Detail: full event timeline, agent/env info |

## 5.5.1 Listing Page Note

The standalone `/sessions` listing has been folded into `/dashboard` — the dashboard's session sidebar is the primary surface for browsing and resuming sessions (see [7.0.0](./7.0.0-dashboard.md)).

## 5.6.0 Session Creation

Required inputs:
- **Agent ID** — which agent to use
- **Environment ID** — which sandbox to run in
- **Agent version** (optional) — pin to specific version

Sessions are created via the dashboard's new-chat form or the sessions API directly.

### 5.6.1 Automatic Vault Attachment

`POST /api/sessions` attaches the caller's Anthropic vault to `vault_ids` on the way through so any MCP credentials the user has stored via [Connections](./10.0.0-connections.md) are available to the session. Caller-supplied `vault_ids` (e.g. from admin overrides) are preserved; the user's vault is de-duplicated if it's already in the list. Vault-lookup failures are logged but do not block session creation — a session without MCP credentials is still useful.
