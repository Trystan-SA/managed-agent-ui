# 3.0.0 — Agent Management

## 3.1.0 Overview

Agents are Claude model configurations stored in the Anthropic API. This domain provides CRUD operations and local bookmarking for organization.

## 3.2.0 Data Model

### 3.2.1 Anthropic-Owned (Source of Truth)

Agents live in the Anthropic API with fields: id, name, model, system prompt, tools configuration, version history, status (active/archived).

### 3.2.2 Local Bookmarks

| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| user_id | uuid | FK → users |
| agent_id | text | Anthropic agent ID |
| nickname | text | Optional local name |
| pinned | boolean | Default false |
| created_at | timestamp | |

Bookmarks add local metadata without duplicating Anthropic state.

## 3.3.0 API Routes

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/agents` | List all agents from Anthropic |
| POST | `/api/agents` | Create new agent |
| GET | `/api/agents/[id]` | Retrieve single agent |
| PUT | `/api/agents/[id]` | Update agent (creates new version) |
| POST | `/api/agents/[id]/archive` | Archive agent |
| GET | `/api/agents/[id]/versions` | List version history |

## 3.4.0 Pages

| Route | Purpose |
|-------|---------|
| `/agents` | List all agents with status badges |
| `/agents/new` | Create agent form (name, model, system prompt, tools) |
| `/agents/[id]` | Detail view: config, version history, linked sessions |
| `/agents/[id]/edit` | Edit form (updates create new version) |

## 3.5.0 Versioning

Every update to an agent creates a new version in the Anthropic API. The version history endpoint returns all past versions. Sessions can be pinned to a specific agent version.

## 3.6.0 Agent Configuration

- **Model**: Claude model to use (e.g., claude-sonnet-4-20250514)
- **System prompt**: Instructions for the agent
- **Tools**: Tool configurations available to the agent
